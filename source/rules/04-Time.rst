Timelines
------------------------------------

Commanders must keep track of the time required for each activity.
Each month only has 28 workdays and commanders cannot accomplish more than 28 days of work in a single month.

The following activities have time required:

* System transit:

  * Transit to/from jump points - Round up the number of days given by the :doc:`system report </systems/index>`

  * Jumping between systems - 0 days

  * Charging in system - Round up the number of days given by the :doc:`system report </systems/index>`

* Combat:

  * Fighting in scenarios - 7 days

* :doc:`Force Management </rules/02-Force-Management>`:

  * *Repair* - 7 days (can *repair* between scenarios)

  * *Replace* - 14 days (units not available for next scenario)

  * *Train* - 7 days per skill level

  * *Retrain* - 7 days (can *retrain* between scenarios)

  * Wound healing - one point of damage heals each 7 days

* :doc:`Dropship Customization </rules/02-Force-Management>`:

  * Combat bay conversion - 28 days, only 1 at a time

  * Support bay conversion - 28 days, can do multiple at a time
